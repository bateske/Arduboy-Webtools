# Arduboy2 Image Conversion & Rendering Specification

*Comprehensive Technical Reference for Building an Image Converter /
Editor*

This document is intended as an authoritative implementation reference
for an automated coding agent building an Arduboy image converter and
editor.

Primary Goal: - Convert source images (PNG, BMP, etc.) into valid
Arduboy2-compatible C arrays - Generate correct rendering code
suggestions - Support all common Arduboy rendering paths - Enforce
correct byte layout, mask generation, and frame packing

------------------------------------------------------------------------

# 1. Arduboy Display Architecture

Resolution: 128 x 64\
Color depth: 1-bit monochrome

Internally the display buffer is arranged in vertical "pages":

-   Each byte represents 8 vertical pixels
-   Bit 0 (LSB) = top pixel
-   Bit 7 (MSB) = bottom pixel
-   Pages are stacked vertically

Memory index formula (vertical format):

page = y // 8\
bit = y % 8\
index = page \* width + x

Pixel write rule:

buffer\[index\] \|= (1 \<\< bit) // set pixel\
buffer\[index\] &= \~(1 \<\< bit) // clear pixel

This vertical page layout defines the canonical Arduboy bitmap format.

------------------------------------------------------------------------

# 2. Coordinate System

-   Origin (0,0) is top-left
-   X increases right
-   Y increases downward

All conversion logic must follow this coordinate orientation.

------------------------------------------------------------------------

# 3. Image Format Types Supported by Arduboy2

The converter must support the following output formats.

------------------------------------------------------------------------

# 4. drawBitmap() Native Vertical Format

Function:

``` cpp
drawBitmap(x, y, bitmap, w, h, color);
```

Requirements: - Height MUST be multiple of 8 - Data stored in PROGMEM -
Data layout: vertical pages

Byte Count: bytes = width \* (height / 8)

Ordering: For each page (top to bottom): For each x from 0 → width-1:
emit byte

Page 0 = rows 0-7\
Page 1 = rows 8-15\
etc.

Bit rule: Bit N in byte corresponds to pixel at: x coordinate = column y
coordinate = (page \* 8) + N

------------------------------------------------------------------------

## Converter Rules for drawBitmap()

Input image may be any height.

If height % 8 != 0: - Default policy: pad image to next multiple of 8 -
Padding pixels must be OFF (0)

Thresholding: - If input has alpha: alpha == 0 → OFF alpha \> threshold
→ ON - If no alpha: brightness threshold determines ON/OFF

------------------------------------------------------------------------

# 5. drawSlowXYBitmap() Horizontal Format

Function:

``` cpp
drawSlowXYBitmap(x, y, bitmap, w, h, color);
```

Format: - Horizontal row-major - Each byte = 8 horizontal pixels - MSB =
leftmost pixel

Byte Count: bytes_per_row = ceil(width / 8) total_bytes = bytes_per_row
\* height

Bit rule: bit 7 = leftmost pixel bit 0 = rightmost pixel

Used only when necessary. Slower than native vertical format.

------------------------------------------------------------------------

# 6. drawCompressed() (RLE Format)

Function:

``` cpp
drawCompressed(x, y, data, color);
```

Requirements: - Height multiple of 8 - Uses Team A.R.G. RLE encoding -
Typically generated by Cabi tool

Converter Options: - Either integrate Cabi algorithm - Or call external
Cabi process

Transparency requires two arrays: - mask array - image array

Rendering pattern:

``` cpp
arduboy.drawCompressed(x, y, mask, BLACK);
arduboy.drawCompressed(x, y, image, WHITE);
```

------------------------------------------------------------------------

# 7. Sprites Class Format

All sprite image arrays begin with:

\[width\]\[height\]

Height must be multiple of 8.

After header: frame0 bytes frame1 bytes ...

Frame size: frame_bytes = width \* (height / 8)

Frame offset: frame_offset = 2 + (frame_index \* frame_bytes)

------------------------------------------------------------------------

# 8. Sprite Rendering Modes

## 8.1 drawOverwrite()

Behavior: 1 bit → set pixel 0 bit → clear pixel

No transparency.

------------------------------------------------------------------------

## 8.2 drawSelfMasked()

1 bit → set pixel 0 bit → leave unchanged

Simplest transparency.

------------------------------------------------------------------------

## 8.3 drawErase()

1 bit → clear pixel 0 bit → unchanged

------------------------------------------------------------------------

## 8.4 drawExternalMask()

Image array: - begins with width,height

Mask array: - NO width,height prefix - identical frame packing -
frame_bytes per frame

Behavior: mask bit == 1 → pixel becomes image bit\
mask bit == 0 → unchanged

------------------------------------------------------------------------

## 8.5 drawPlusMask()

Array layout:

\[width\]\[height\] (image_byte, mask_byte) (image_byte, mask_byte) ...

Total bytes per frame: frame_bytes = width \* (height / 8)
plusmask_frame_bytes = frame_bytes \* 2

For each image byte: - First byte = image bits - Second byte = mask bits

Recommended default for clean transparency.

------------------------------------------------------------------------

# 9. Frame Extraction Rules

Converter must support:

-   Horizontal strip atlas
-   Grid atlas
-   Single frame

Atlas parameters: - frame width - frame height - columns - rows

Frames are extracted left-to-right, top-to-bottom.

------------------------------------------------------------------------

# 10. Mask Generation Rules

If source image has alpha: - alpha == 0 → mask bit = 0 - alpha \> 0 →
mask bit = 1

If no alpha: - mask bit = image bit

Optional: allow explicit separate mask input.

------------------------------------------------------------------------

# 11. Required Export Targets

The tool MUST support exporting:

1.  drawBitmap format
2.  drawSlowXYBitmap format
3.  Sprites image only
4.  Sprites + external mask
5.  Sprites + plus-mask
6.  drawCompressed (RLE)

------------------------------------------------------------------------

# 12. Code Generation Templates

Example template for Sprites:

``` cpp
#include <Arduboy2.h>
#include <Sprites.h>

const uint8_t PROGMEM sprite[] = {
  width, height,
  ...
};

sprites.drawSelfMasked(x, y, sprite, frame);
```

Example template for drawBitmap:

``` cpp
Arduboy2Base::drawBitmap(x, y, bitmap, w, h, WHITE);
```

------------------------------------------------------------------------

# 13. Validation Tests for Converter

The tool MUST validate bit correctness using synthetic patterns:

Test images: - Diagonal line - Checkerboard - 8x8 quadrant test - Single
pixel at (0,0), (7,7), etc.

Expected byte values must be verified automatically.

------------------------------------------------------------------------

# 14. Performance Considerations

Fastest: - drawBitmap - Sprites (vertical format)

Slower: - drawSlowXYBitmap - drawCompressed

Prefer vertical format whenever possible.

------------------------------------------------------------------------

# 15. Implementation Checklist for Agent

When generating Arduboy output:

1.  Enforce height multiple-of-8 (pad if needed)
2.  Use correct vertical bit packing
3.  Validate frame byte count
4.  Validate mask size consistency
5.  Generate correct PROGMEM declarations
6.  Generate matching draw call snippet
7.  Provide memory usage summary
